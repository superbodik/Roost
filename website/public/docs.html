Install & update

One command, Ubuntu 22.04/24.04, run as root.

bash <(curl -sSL https://raw.githubusercontent.com/superbodik/PowerNode/main/install.sh)

The installer asks which language to use, then offers a menu: install the master panel, install a daemon node, install both on one host, or uninstall. Picking the panel also asks for a domain (for a free Let's Encrypt certificate) and creates your first admin account — there is no signup form, only panel-admin and this installer create accounts.

To update an existing install, run the same command with a flag instead of picking from the menu:

PANEL_UPDATE=1 bash <(curl -sSL https://raw.githubusercontent.com/superbodik/PowerNode/main/install.sh)

Adding a node

Create the node first in the panel (Nodes → Add node) — it'll show you a ready-to-paste one-liner with the daemon token and panel URL already filled in. Run that exact command on the node's server instead of picking option 2 from the interactive menu by hand; the one-liner sets WINGSD_PANEL_URL for you automatically, which the interactive path can't always guess (see the SFTP section below for what happens if it's missing).

SFTP: "Authentication required" / "No more authentication methods available"

By far the most common support question. Here's the full decision tree.

1. SFTP here is public-key only — there is no password login

If your client is set to log in with a username/password (FileZilla's Quick Connect bar does this by default), it will always fail with exactly this error, because the server never offers a password method at all. You need a registered SSH key.

  1. Generate a key pair if you don't have one (ssh-keygen -t ed25519, or PuTTYgen on Windows).
  2. Paste the public key into Account → SSH Keys in the panel.
  3. In your SFTP client, use the matching private key file to log in — not a password.
Windows + FileZilla
FileZilla's SFTP engine needs a PuTTY-format .ppk key, not the raw OpenSSH key ssh-keygen produces. Open PuTTYgen → Conversions → Import key, load your OpenSSH private key, then Save private key as .ppk and point FileZilla's Site Manager (Logon Type: Key file) at that file. Prefer to skip the conversion entirely? Use WinSCP or Windows' built-in sftp/ssh client instead — both accept the OpenSSH key directly.

2. Username format

It's <your panel username>.<server id> — both are shown in the SFTP access card on the server's Overview tab. Port is always 2022.

3. Node still refuses every key ("no auth passed yet")

If the key and username are both correct and it's still failing, the node itself is usually missing its panel address. The SFTP server on each node has to call back to the panel to verify every login, and if it doesn't know where the panel is, it rejects the connection before it even looks at your key or username.

Check this first
On the node, run: grep PANEL_URL /etc/wingsd/wingsd.env. If it's empty or missing, that's the bug. Fix it and restart: 
echo "WINGSD_PANEL_URL=http://127.0.0.1:8080" >> /etc/wingsd/wingsd.env
systemctl restart wingsd
Use 127.0.0.1:8080 only if the panel runs on this same server (an all-in-one install). If the panel is on a different host, use its real address instead, e.g. https://panel.example.com.

Newer installs set this automatically — this only bites installs done through the interactive menu (option 2, node-only) before that fix, or manual reinstalls that skipped the prompt.

4. Still stuck? Read the real reason

SSH deliberately never tells the client why a key was rejected (that's normal protocol behavior, not a bug) — but the node logs the exact reason:

journalctl -u wingsd -f

Try connecting again and watch for a line like sftp: auth failed for <ip>: ... — it'll say exactly what went wrong: unknown user, unrecognized key, no access to that server, or the missing-panel-URL error above.

Custom domains & HTTPS not issuing

Attaching a domain to a server (or to the panel itself during install) runs certbot against Let's Encrypt, which only works if DNS is already correct.

  • Make sure the domain's A record already points at the node's public IP before you add it — certbot can't issue a certificate for a domain that doesn't resolve to this server yet.
  • If it fails, the domain stays on plain HTTP instead of blocking the whole operation — fix DNS, then remove and re-add the domain (or re-run the panel installer's domain step).
  • Both the panel install and the per-server domains feature use the same certbot/nginx setup on the node, so the fix is the same either way.

Common install errors

PostgreSQL cluster broken after a full uninstall + reinstall

If you ran the full removal option and then reinstalled, and Postgres fails to start or the panel can't reach the database, apt likely left a partial PostgreSQL install behind (some installs only purge the postgresql metapackage, not the actual versioned engine like postgresql-16). The installer handles this correctly on current versions — if you're still hitting it, purge every leftover postgresql* package by name and clean the config directories before reinstalling:

dpkg -l | grep postgresql
apt-get purge -y $(dpkg -l | grep postgresql | awk '{print $2}')
rm -rf /etc/postgresql /etc/postgresql-common /var/log/postgresql

Then re-run install.sh.

"Docker must be installed before wingsd"

The daemon needs Docker to actually run game server containers. The installer installs it automatically on supported Ubuntu versions — if this still shows up, Docker's own install script failed silently (usually a broken apt source). Check docker --version manually and install Docker per Docker's own instructions if needed, then re-run install.sh.
